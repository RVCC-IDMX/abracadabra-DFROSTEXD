Essential settings

VS Code works great out of the box, but these settings will optimize your experience for the Abracadabra project:

Accessing settings

  1. Open settings: File ‚Üí Preferences ‚Üí Settings (or Ctrl+, / Cmd+,)
  2. Search for settings: Type in the search box to find specific options
  3. Toggle between UI and JSON: Click the {} icon for advanced editing

Recommended settings for your project

Format on save

Automatically cleans up your package.json and other files when you save:

"editor.formatOnSave": true

Tab size for web development

Consistent 2-space indentation for JSON, HTML, and CSS:

"editor.tabSize": 2,
"editor.insertSpaces": true

Auto save

Prevents losing work when testing npm scripts:

"files.autoSave": "afterDelay",
"files.autoSaveDelay": 1000

Troubleshooting

Common Abracadabra project issues

‚ùå Terminal not in project directory

Problem: npm commands fail because terminal opens in wrong location

Solution:

  1. Close VS Code completely
  2. Open project: File ‚Üí Open Folder ‚Üí select abracadabra-starter
  3. Terminal should now open in project root
  4. Verify with pwd command

‚ùå Package.json IntelliSense not working

Problem: No auto-complete for npm scripts or dependencies

Solution:

  1. Verify file is saved as package.json (not package.txt)
  2. Check for JSON syntax errors (missing commas, quotes)
  3. Open entire project folder, not just the file
  4. Restart VS Code if IntelliSense still not working

‚ùå Live Server not appearing in context menu

Problem: "Open with Live Server" option missing when right-clicking HTML files

Solution:

  1. Install Live Server extension (check Extensions panel)
  2. Right-click on index.html file specifically (not folder)
  3. Ensure file has .html extension
  4. Try Developer ‚Üí Reload Window

‚ùå npm commands not found in terminal

Problem: "npm: command not found" or similar error

Solution:

  1. Install Node.js from nodejs.org
  2. Restart VS Code after Node.js installation
  3. Verify installation: run node --version and npm --version
  4. Ensure you're in the correct project directory

Performance optimization

üí° VS Code performance tips

  • Extension management: Disable unused extensions to improve startup time
  • File exclusions: Exclude node_modules from search (already configured in Abracadabra)
  • Auto-save balance: Don't set auto-save too frequently if using slow file watchers
  • Memory management: Restart VS Code if it becomes sluggish during development

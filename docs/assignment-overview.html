2. npm - The Package Manager

Think of npm like an app store for code. Instead of downloading games, you download useful code tools.

What it does:

  • Manages your project's dependencies (tools your project needs)
  • Runs automated tasks (like starting a web server)
  • Keeps track of your project's configuration

Real example: Instead of manually setting up a web server, npm can do it for you with one command.

Think of it like a recipe:

  • Contains "cooking instructions" (scripts) for common tasks

Key Concepts Explained Simply

What's a "port"?

Think of ports like different doors to your computer. Port 3000 is just a specific door where your website will be available. When you type localhost:3000 in your browser, you're saying "show me what's behind door 3000."

What's "localhost"?

This is just a special address that means "my own computer." So localhost:3000 means "show me what's on my computer, through door 3000."

Why does the magic only work on port 3000?

The JavaScript code is smart! It checks what port you're using and only shows the transformed version on port 3000. This teaches you about environment detection - making your code behave differently in different situations.

What are npm scripts?

These are shortcuts you can create to run complex commands easily. Instead of remembering long commands, you can just type npm run dev and it runs everything you need.
